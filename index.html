Server Connection and Security Settings
In this section, we highlight the steps for server connection settings and the importance of security in operational environments. These settings include the following:
1. Server Connection Settings:
Installing SSH: Ensure that the SSH service is installed on the server using the following command:
sudo apt-get install openssh-server
2. Server Connection Methods:
Read More

There are two main ways to connect to the server:
Using Keys:
Generate the keys using the command:
ssh-keygen -t rsa -b 4096
Copy the public key to the remote server:
ssh-copy-id user@remote_server
Using Username and Password:
It is recommended to use strong and complex usernames and passwords to ensure the security of the server connection.
3. Connecting to the Server:
 Server Address: Specify the correct IP address or domain name of the server to ensure an effective connection, such as:
ssh user@192.168.1.100
Connection Port: Make sure to use the correct port (e.g., 22 for SSH or 3306 for MySQL) to access the desired service. You can specify the port during the connection:
ssh -p 22 user@remote_server
4. Accepting the Fingerprint:
When connecting to the server for the first time, you will be prompted to accept the server’s fingerprint to verify the server's identity. You should verify the fingerprint and match it with the known one.
5. sshd_config File Settings:
This file includes important settings for the SSH server and can be modified to enhance the security of the connection. The most important settings are:
Changing the Default Connection Port:
Port 2222
(It is recommended to change the default port 22 to another number for enhanced security).
Disabling Root Login:
PermitRootLogin no
(To prevent direct login as the root user).
Allowing Specific IP Addresses:
AllowUsers user1@192.168.1.100 user2@192.168.1.200
(To specify certain IP addresses that are allowed to connect to the server).
Using Public Key Authentication Only:
PasswordAuthentication no
(To disable password authentication and allow keys only).
Specifying the Protocol Used:
Protocol 2
(To use the more secure SSH-2 protocol).
Setting the Maximum Number of Failed Attempts:
MaxAuthTries 3
(To specify the number of authentication attempts before blocking the connection).
Session Settings:
Closing the connection after a period of inactivity:
ClientAliveInterval 300
ClientAliveCountMax 0
(This sets the connection to close after 5 minutes of inactivity).
After configuring these options and modifying the sshd_config file, make sure to restart the SSH service to apply the changes:
sudo systemctl restart ssh
These settings enhance the security of the server connection and reduce the risk of unauthorized access.
